## 计划

### 1. 本阶段定位

P5 面向的是“系统控制力”。这是从产品工作台走向内部操作系统层的关键阶段。

### 2. 交付范围

- 微短剧 11 节点
- 9 Agent + 协调器
- 全屏 Pipeline
- Debug 面板
- 节点嵌套 / 子流程
- 数据反馈闭环基础
- 更完整的失败恢复和续跑能力

### 3. 阶段结果定义

Ultimate 用户应该能看到、理解、干预复杂流程，而普通用户仍不需要承受这种复杂度。

## 测试

### 1. 先写什么

- 协调器调度测试

- 失败恢复与续跑测试

### 2. 压力测试

- 长链路 pipeline 运行

- 中途失败恢复
- debug 数据采样与落库

## 验证

### 1. 验收标准

- 9 Agent 协调可观测
- 全屏 Pipeline 可以诊断复杂链路
- 单节点失败后能续跑或恢复
- Debug 数据可用于定位成本、时延、错误

### 2. 退出门槛

- `Ultimate` 具备内部调试和高级实验价值
- 节点嵌套不破坏主数据流
- 失败恢复不是演示能力，而是可复现可验证能力

## 修复闭环

### 1. 高频问题池

- 协调器重复调度
- 嵌套流程卡死
- Debug 指标缺失或错误
- 失败恢复后二次污染

### 2. 修复策略

1. 先抓 trace 和 run_id
2. 复现失败链路
3. 写回归测试
4. 修编排与状态机
5. 回归恢复链路与 metrics

### 3. 不允许带入 P6 的问题

- 全屏 Pipeline 只能看不能排障
- 9 Agent 无法稳定长期运行
- 失败恢复不可信
