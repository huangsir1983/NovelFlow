# 画布节点数据传递增强：完善 Pipeline 全链路信息流

## Context

当前造物画布 Pipeline（Scene → Shot → PromptAssembly → ImageGeneration → VideoGeneration）的数据传递存在严重不足：
- `PromptAssemblyNode.characterRefs` 始终为空数组 `[]`，角色视觉参考完全未传入
- Shot 的 `visual_prompt` 在脚本生成后为空，导致 PromptAssembly 和 ImageGeneration 的 prompt 也为空
- 场景的 `characters_present`、角色的 `appearance`/`costume`/`visual_reference` 未流入下游
- 地点的 `visual_description`/`mood`/`lighting` 等丰富数据未传入 PromptAssembly
- `VideoGenerationNode.durationMs` 始终为 0
- 场景情绪/叙事模式等风格数据未参与 prompt 组装

**目标**：让每个节点携带足够的结构化数据，使 PromptAssembly 能自动组装出完整的视觉 prompt，无需用户手动填写。

**3.3 新增 `assembleVisualPrompt` 工具函数**

当 `shot.visualPrompt` 为空时，自动从结构化数据组装 prompt：

```typescript
function assembleVisualPrompt(
  shot: ShotInput,
  scene: SceneInput,
  characterRefs: CharacterRefInfo[],
  locationDetail?: LocationDetail,
): string {
  const parts: string[] = [];
  // 1. 镜头构图: "close-up shot, low angle, dolly in"
  if (shot.framing) parts.push(shot.framing);
  if (shot.cameraAngle && shot.cameraAngle !== shot.framing) parts.push(shot.cameraAngle);
  // 2. 核心动作描述
  if (shot.description) parts.push(shot.description);
  // 3. 角色外观（仅 in-frame 的）
  for (const c of characterRefs) {
    if (c.appearanceDesc) parts.push(`${c.name}: ${c.appearanceDesc}`);
  }
  // 4. 环境/地点
  if (locationDetail?.visualDescription) parts.push(locationDetail.visualDescription);
  else if (scene.location) parts.push(scene.location);
  // 5. 光照
  if (locationDetail?.lighting) parts.push(locationDetail.lighting);
  else if (scene.timeOfDay) parts.push(scene.timeOfDay);
  // 6. 情绪氛围
  if (scene.emotionalPeak) parts.push(`mood: ${scene.emotionalPeak}`);
  if (locationDetail?.mood) parts.push(`atmosphere: ${locationDetail.mood}`);
  // 7. 运镜
  if (shot.cameraMovement) parts.push(`camera: ${shot.cameraMovement}`);
  return parts.join(', ');
}
```

**3.4 在 shot 循环中构建 characterRefs**

```typescript

const shotCharNames = shot.charactersInFrame?.length
  ? shot.charactersInFrame
  : scene.characterNames; // fallback 到场景所有角色

const characterRefs: CharacterRefInfo[] = shotCharNames
  .map(name => characterMap?.[name])
  .filter(Boolean)
  .map(c => ({
    name: c.name,
    visualRefUrl: c.visualRefUrl,
    visualRefStorageKey: c.visualRefStorageKey,
    appearanceDesc: c.appearance
      ? [c.appearance.face, c.appearance.body, c.appearance.hair, c.appearance.distinguishing_features]

const locationDetail = locationDetailMap?.[scene.location];
const locationPanorama = locationPanoramaMap?.[scene.location];

const autoPrompt = shot.visualPrompt || assembleVisualPrompt(shot, scene, characterRefs, locationDetail);
```

## 风险与应对

1. **`characterRefs` 类型变更**：从 `string[]` → `CharacterRefInfo[]` 是破坏性改动。PromptAssemblyNode.tsx 中 `data.characterRefs.map(n => ...)` 需同步修改。在同一 commit 中完成。
2. **hash 性能**：character/location 数据加入 hash 可能导致不必要的重建。只 hash ID 列表和关键标记，不 hash 全量文本。
3. **自动 prompt 质量**：`assembleVisualPrompt` 是基于模板拼接的 MVP 方案。后续可升级为 LLM 调用（已有 `CanvasAgentTaskType: 'generate-image-prompt'`）。
4. **数据量**：角色 appearance 对象传入每个节点会增加内存。通过 `charactersInFrame` 限定只传入镜头中实际出场的角色（而非场景所有角色）来控制。
