# 前端计划：适配 R29 后端双流流水线

## 背景

后端 `import_pipeline.py` 已完成 R29 迁移，核心变化：

| 变化 | 旧后端 | 新后端 (R29) |
|------|--------|-------------|
| 提取模式 | 单流合并 (combined) | 双流分离 (char_stream + scene_stream 并行) |
| PHASES | `streaming → locations → props → variants → knowledge → shots → merging → prompts` | `streaming → enrichment → knowledge → shots → merging → prompts` |
| enrichment | 顺序执行 locations → props → variants | 并行: locations(渐进) + props + variants，嵌入 streaming 阶段 |
| 新 SSE 事件 | 无 | `character_found` / `scene_found` 在 streaming 阶段实时到达；`location_card` 在 enrichment 阶段到达 |
| phase 名称 | `locations` / `props` / `variants` 各自独立 | 合并为 `enrichment` 一个 phase |

前端当前状态：`PHASE_LABELS` 里根本没有 `streaming` / `locations` / `props` / `variants` / `enrichment`——
它用的是一套旧的 phase 名（`segmenting` / `scenes` / `characters`），与后端实际发出的 SSE `phase` 字段**完全不匹配**。

### 4. 增强 SSE handler — `page.tsx` (line 169-264)

#### 4a. 处理 `character_found` 事件（后端新发的）

后端 `_stream_characters` 在每个角色提取后会 emit：
```python
{"type": "character_found", "phase": "streaming", "data": {"name": ..., "role": ..., "index": ...}}
```

在 SSE handler 的 switch 中新增 case：

```typescript
case 'character_found': {
  const name = (data as any).data?.name as string;
  if (name) {
    setImportEntities((prev) => [...prev, name]);
  }
  setStreamingStats((prev) => {
    const newStats = { ...prev, chars: (prev.chars || 0) + 1 };
    updateStep('streaming', {
      status: 'running',
      detail: `角色: ${newStats.chars} | 场景: ${newStats.scenes}`,
    });
    return newStats;
  });
  break;
}
```

#### 4b. 处理 `scene_found` 事件

后端 `_stream_scenes` 在每个场景提取后会 emit：
```python
{"type": "scene_found", "phase": "streaming", "data": {"scene_id": ..., "location": ..., "core_event": ..., "index": ...}}
```

```typescript
case 'scene_found': {

    const newStats = { ...prev, scenes: (prev.scenes || 0) + 1 };
    updateStep('streaming', {
      status: 'running',
```

### 5. 扩展 ImportSSEEvent 类型 — `project.ts` (line 177-185)

新增三个事件类型：

```typescript
export type ImportSSEEvent =
  | { type: 'phase_start'; phase: string }
  | { type: 'phase_done'; phase: string; data?: Record<string, unknown> }
  | { type: 'character_found'; phase: string; data: { name: string; role: string; index: number } }
  | { type: 'scene_found'; phase: string; data: { scene_id: string; location: string; core_event: string; index: number } }
  | { type: 'location_card'; phase: string; data: { name: string; scene_count: number; index: number; total: number } }
  | { type: 'prop_card'; phase: string; data: { name: string; category: string; index: number; total: number } }
  | { type: 'variant'; phase: string; data: { character: string; variant_count: number } }
  | { type: 'warning'; phase: string; data: { message: string; character?: string } }
  | { type: 'item_ready'; phase: string; sub?: string; data?: Record<string, unknown> }
  | { type: 'chapter_progress'; phase: string; index: number; total: number; beats?: number; scenes?: number }
  | { type: 'window_progress'; phase: string; index: number; total: number; scenes_in_window?: number }
  | { type: 'scene_progress'; phase: string; index: number; total: number; shots?: number }
  | { type: 'pipeline_complete'; summary: Record<string, number> }
  | { type: 'error'; message: string; phase?: string; retryable?: boolean };
```

### 6. 增强 ImportProgress 组件 — `ImportProgress.tsx`

#### 6a. streaming step 的实时实体标签

**现状**：`entities` 数组在 characters phase 下显示角色名标签。

**调整**：在 `streaming` step 运行时也显示实体标签（角色名），逻辑不变，因为 `importEntities` 仍然在 `character_found` 时被 push。无需改动组件本身，只需确保 entities prop 持续传入。

#### 6b. 可选增强：子步骤展开

streaming step 可展开显示双流细节：
```
✓ 上传完成
⟳ 双流提取           角色: 8 | 场景: 32
    ├─ 角色流 ✓ (8个角色)
    └─ 场景流 ⟳ (32/~40 场景)
⟳ 资产丰富化         位置卡: 3/5
    ├─ 位置卡 ⟳ (3/5)
    ├─ 道具卡 ✓
    └─ 变体 ⟳ (高明远: 4个)
○ 知识库构建
○ 分镜拆解
○ 镜头合并
○ 视觉提示词生成
```

**实现方式**：为 `ImportStep` 接口增加可选 `subSteps` 字段：

```typescript
interface ImportStep {
  id: string;
  label: string;
  status: 'pending' | 'running' | 'done' | 'error';
  detail?: string;
  progress?: { current: number; total: number };
  subSteps?: { label: string; status: 'pending' | 'running' | 'done' }[];  // 新增
}
```

在 `ImportProgress` 渲染中，当 `subSteps` 存在时缩进渲染子行。

**优先级**：低。首轮可只改 PHASE_LABELS + SSE handler，子步骤展开作为第二轮增强。

## 风险与注意

- **后端 fallback 路径**：当双流失败回退到 `_run_streaming_extraction`（旧单流）时，后端发的 phase 名仍然是 `streaming`，前端不需要特殊处理
- **旧 phase 名兼容**：如果数据库中有旧任务的 `current_phase` 存为 `locations` / `props` / `variants`，resume 时 `start_phase_idx` 会找不到匹配——后端 `PHASES.index()` 会抛异常。**建议后端加一个兼容映射**：`{"locations": 1, "props": 1, "variants": 1}` → 都映射到 `enrichment` 索引
- **i18n**：`PHASE_LABELS` 目前硬编码中文，未来应迁移到 `tImport()` 翻译函数。首轮可保持硬编码
