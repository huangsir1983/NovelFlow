> 版本: 7.0 | 日期: 2026-03-06

## 一、架构总纲

### 1.1 核心范式转换

```
v6.0 架构 (旧):
  节点画布 (一个界面承载一切)
  └── Normal: 向导式锁定画布
  └── Canvas+: 自由节点画布
  └── Hidden+: + Cinema Lab

v7.0 架构 (新 — 三段式创作引擎):
  ┌──────────┐    ┌──────────┐    ┌──────────┐
  │ 📝 工坊   │ →  │ 🎨 画板   │ →  │ 🎬 预演   │
  │ Workbench │    │ Board    │    │ Preview  │
  │ 文学创作   │    │ 视觉编排  │    │ 时序预演  │
  └──────────┘    └──────────┘    └──────────┘
       │               │               │
  Sudowrite式      TapFlow式创作     Premiere式
  文档流编辑器      无限可执行画布     时间线+监视器
       │               │               │
  语言思维模式      空间思维模式      时间思维模式

  + Pipeline面板 (抽屉式，Canvas+可见，节点画布沉为引擎盖)
  + 知识引擎 (三段全程嵌入，被动建议)
```

### 1.2 为什么不是节点画布

| 维度 | 节点画布 (旧) | 三段式 (新) |
|------|-------------|------------|
| 认知模式 | 系统架构思维 (工程师) | 语言→空间→时间 (创作者) |
| 启动到首次有意义操作 | 5-10分钟 (理解节点→拖→连→配置) | 30秒 (选意图→上传) |
| 连线操作频率 | 每次加节点都要手动连 | 零 (系统自动推断关系) |
| 认知负载 | "这个端口连哪里？" | "下一步该审阅什么？" |
| 画布时间 vs 创作时间 | 50:50 | 20:80 |
| 学习曲线 | 陡峭 (需理解节点系统) | 近乎零 (意图驱动) |
| 专业用户自由度 | 有 | 仍然有 (Pipeline面板渐进披露) |

### 1.3 核心设计决策

```
决策1 (v7.0新增): 三段式替代节点画布
  三种认知模式 × 三种最佳界面范式
  节点画布沉为"引擎盖"(Pipeline面板)，不再是主界面
  基本操作单元: Shot Card (镜头卡片)，不是 Node (节点)

决策2 (v7.0新增): Stage 2 采用 TapFlow式创作画板
  7维决策矩阵加权评分: TapFlow式 8.00 > Figma/Miro式 5.75 > ComfyUI式 5.40
  空间邻近性 = 语义关联 (参考图放在Shot旁边 = 创作意图)
  卡片可执行 (点击按钮原地生成，不需要连线)
  两层模型: 创作视图 (默认，无连线) + 流程视图 (Canvas+，显示连线)

决策3 (v7.0新增): 知识引擎全程嵌入
  不是独立模块，而是三段每个操作点的被动建议
  工坊: 叙事结构参考 / 画板: 视觉语言参考 / 预演: 节奏声画参考

决策4 (保留): 资产库 — 跨项目可复用素材管理系统
  三层素材仓库: 官方公用 + 私人分享 + 自用 + 项目内
  锁定机制 → 下游视觉生成强制使用参考图 (一致性锚点)

决策5 (保留): 后期制作交给剪映
  系统输出剪映工程包 (Draft文件夹)
  包含: 视频片段 + TTS音频 + AI音乐 + 字幕SRT + 时间线草稿 + 导演标记

决策6 (保留): AI音乐 (Suno/Udio) 替代纯标注
  声音叙事 → 可用音频

决策7 (保留): PostgreSQL from Phase 0
  Docker Compose, 跳过 SQLite
```

## 三、Stage 1 — 工坊 (Workbench) 详细设计

### 3.1 核心理念

> 用户面对的不是节点，而是自己的文字。AI像随叫随到的编剧搭档。

**认知模式**: 语言思维
**界面范式**: Sudowrite式文档流编辑器
**基本操作单元**: Beat / Scene (叙事块)
**用户心流**: 遣词造句 → 情感表达 → 叙事节奏

### 3.6 AI操作清单 (Sudowrite式)

| 操作 | 触发方式 | 功能 | 版本 |
|------|---------|------|------|
| **续写** | 段落末尾 → 弹出 | AI给出3种续写方向 | Canvas+ |
| **扩写** | 选中文本 → 右键/工具栏 | 用五感补充细节 | Canvas+ |
| **改写** | 选中文本 → 右键/工具栏 | 保持含义，换表达方式 | Canvas+ |
| **精简** | 选中文本 → 右键/工具栏 | 压缩保留核心 | Canvas+ |
| **视觉化** | 选中文本 → 右键/工具栏 | 补充视觉/听觉/触觉描写 | Canvas+ |
| **换语气** | 选中文本 → 右键/工具栏 | 更紧张/更平静/更幽默等 | Canvas+ |
| **建议** | 自动 (AI侧边栏) | 实时分析当前内容给建议 | Normal+ |
| **角色语调** | 对白处 → 工具栏 | 确保对白符合角色性格 | Hidden+ |
| **张力优化** | Beat元数据 → 点击 | 分析并优化当前Beat张力 | Canvas+ |

### 3.7 Normal版自动推进流程

Normal版用户不需要自己做任何"搭建"工作:

```
Step 1: 上传小说/剧本 (或选择空白模板)
  → AI自动: 格式识别 → 章节拆分 → 角色提取 → 场景提取 → Story Bible初始化
  → 用户看到: 左侧导航已自动填充完毕
  → 用户操作: 审阅，确认或修改

Step 2: AI自动生成Beat Sheet
  → 用户看到: 每章的Beat列表已填好
  → 用户操作: 阅读，微调个别Beat

Step 3: AI自动生成Scene (剧本)
  → 用户看到: 每个Beat展开为完整的Scene
  → 用户操作: 阅读，微调对白和描写

Step 4: 工坊自动建议"进入画板"
  → 底部出现: "✅ Beat和Scene已稳定，建议进入画板查看分镜 [进入画板 →]"
  → 用户点击 → 切换到Stage 2
```

### 3.8 Stage间过渡

```
工坊 → 画板:
  触发: Beat+Scene稳定 + 用户主动切换 / 系统建议
  数据: Scene自动生成Shot草稿 → 画板中出现分镜卡片
  保持: 工坊内容不丢失，随时可以切回修改

画板 → 预演:
  触发: Shot+素材链就绪 + 用户主动切换 / 系统建议
  数据: Shot自动填充到时间线轨道
  保持: 画板状态不丢失

任意阶段 → 任意阶段:
  非线性跳转: 预演中发现对白问题 → 点击Shot → 跳回工坊对应Beat
  上下文保持: 每个Stage记忆上次离开时的位置和状态
```

- **高冲击静默**: 爆发前的1-2秒完全静默 → 放大后续冲击
- **过渡静默**: 场景切换时的呼吸空间
- **叙事静默**: 角色内心转折时的留白

### 7.5 声音转场

- **J-Cut**: 声音先于画面切换 (先闻其声)
- **L-Cut**: 声音延续到下一个画面 (余音绕梁)
- **声音桥**: 声音连续跨越画面切换
- **音量渐变**: 渐进式过渡

  → 返回3个候选音频

### 13.4 Hook触发体系

**管线Hook (Pipeline)**:
```

```

**内容Hook (Content)**:
```

角色资料修改            → 一致性Agent (关系检查)

资产锁定变更            → 标记下游Shot + 触发重生成
```

**定时Hook (Schedule)**:
```
每30分钟 (活跃项目) → 一致性Agent (全量扫描)
每小时               → 张力模块 (偏差检测)
每日                 → 数据优化Agent (拉取数据)
```

**Agent间Hook (Inter-Agent)**:
```
编剧 (Scene完成)     → 一致性 (校验)
一致性 (冲突)        → 编剧 (修正)
导演 (分镜完成)      → Visual (Prompt)

## 十六、知识引擎

### 16.1 核心理念

> 知识引擎不是独立模块，而是三段式每个操作点的被动建议层。
> 不替用户做决策，但确保每个决策都有大师级参考。

### 16.2 三段嵌入方式

**Stage 1 工坊** — 叙事知识:
```
用户写 "他恐惧地看着天空"
→ 知识引擎: "考虑用行动而非情感词表达恐惧。
   参考:《教父》开场5分钟没有一个情感形容词，
   全靠肢体语言和停顿传递不安。
   建议改写: '他的手指停在栏杆上，指节发白。'"
```

**Stage 2 画板** — 视觉知识:
```
用户拖一个"全景"到Shot 001
→ 知识引擎: "本场戏推荐以全景开场 ✓
   但注意: 如果前一场结尾也是全景，
   观众会感到视觉重复。Sc11结尾是[中景]，
   景别跳跃合理。✅"
```

**Stage 3 预演** — 节奏知识:
```
用户播放到Shot 005-006接缝
→ 知识引擎: "⚡ 节奏建议
   当前: 两个3s中景连续，注意力可能下降。
   方案A: 插入0.8s反应特写打破节奏。
   方案B: 缩短005到1.5s，利用不对称时长制造不安。
   参考:《寄生虫》地下室发现段落节奏图谱。"

### 17.2 7种资产类型

| 类型 | 内容 | 用途 |

### 17.3 锁定机制详解

```
资产锁定流转:
  角色视觉卡创建 → 参考图上传/AI生成 → 用户确认
  → 点击 🔒 锁定
  → 锁定后:
    · 卡片边框变金色
    · 下游所有引用此角色的Shot:
      · Visual Prompt生成时强制嵌入参考图
      · 生成结果必须与参考面保持一致
      · 一致性守护自动检测偏差
    · 解锁需要: 用户手动解锁 (团队模式下需确认)

资产变更传播:
  锁定资产修改 (如换发型)
  → 系统自动标记所有引用Shot为 ⚠
  → 用户选择: [全部重新生成] 或 [逐个确认]
  → 一致性守护重新扫描
```

## 二十、Animatic动态预览

### 20.1 定位

Stage 3预演台的核心播放引擎。介于分镜和最终成片之间的"动态草稿"。

### 20.2 技术实现

```
输入:
  Shot分镜图 (静态) + Shot参数 (景别/运镜/时长/转场)
  + TTS音频 + AI音乐 + 音效 + 字幕

处理:
  静态图 → Ken Burns动效 (推/拉/平移, 基于运镜参数)
  相邻Shot → 转场效果 (14种)
  音频轨 → 多轨混合
  字幕 → 时间轴同步叠加

输出:
  实时预览播放 (Stage 3监视器)
  可导出为视频文件 (MP4)
  可导出为剪映Draft
```

### 24.3 核心数据模型

```sql
-- 项目 (现有, 扩展)
Project (
  id UUID, name, description,
  import_source VARCHAR,   -- 'novel' | 'script' | 'blank'
  edition VARCHAR,         -- 'normal' | 'canvas' | 'hidden' | 'ultimate'
  stage VARCHAR,           -- 'import' | 'analysis' | 'writing' | 'boarding' | 'preview' | 'export' | 'complete'
  current_stage VARCHAR,   -- ★ 'workbench' | 'board' | 'preview' (三段式当前Stage)
  ...timestamps
)

-- 内容层级
Chapter (project_id, title, order, content TEXT)
Beat (chapter_id, title, order, content TEXT, emotion VARCHAR, tension_score DECIMAL)
Scene (beat_id, title, order, content TEXT, location_id, characters UUID[])
Shot (
  scene_id, order, scale VARCHAR, angle VARCHAR, movement VARCHAR,
  focal_length INTEGER, duration_seconds DECIMAL, transition VARCHAR,
  thumbnail_url VARCHAR, status VARCHAR,
  cinema_lab JSONB,            -- Cinema Lab完整参数
  sound_narrative JSONB,       -- 声音叙事
  mise_en_scene JSONB,         -- 场面调度
  tension_score DECIMAL,
  visual_motifs JSONB,
  cultural_preset VARCHAR,
  thrill_type VARCHAR,
  thrill_visual_strategy JSONB,
  director_notes TEXT
)

-- 知识库
KnowledgeBase (project_id, story_bible JSONB, world_rules JSONB, style_guide JSONB)
Character (project_id, name, description, psychology JSONB, arc JSONB, relationships JSONB)
Location (project_id, name, description, visual_style JSONB, variants JSONB)

-- Pipeline (替代WorkflowCanvas)
PipelineRun (
  id UUID, project_id UUID,
  template_id VARCHAR,
  status VARCHAR,             -- 'pending' | 'running' | 'completed' | 'failed'
  nodes JSONB,                -- 节点配置 (Pipeline面板用)
  edges JSONB,
  progress DECIMAL,
  started_at, completed_at
)
NodeExecution (
  pipeline_run_id, node_id, node_type, status, progress,
  input_snapshot JSONB, output_snapshot JSONB,
  agent_task_id UUID, tokens_used INTEGER, model_used VARCHAR,
  duration_ms INTEGER, error TEXT
)

-- Marketplace
MarketplaceWorkflow (
  slug, creator_id, name, category, tags[],
  required_edition, nodes JSONB, edges JSONB,
  preview_images[], clone_count, rating, forked_from
)

-- 协作
Team (id, name, owner_id)
Comment (project_id, shot_id, user_id, content, position JSONB, resolved BOOLEAN)
Activity (project_id, user_id, action, target, timestamp)
```

## 二十九、风险与缓解

### 29.1 P0 — 关键风险

| 风险 | 影响 | 缓解 |
|------|------|------|
| TipTap+Yjs工坊编辑器性能 | 大文档(>5万字)卡顿 | 虚拟化渲染+分章加载+debounce保存 |
| 画板大量Shot Card渲染 | >200卡片时帧率下降 | 虚拟化(只渲染可视区)+缩放时LOD |
| Animatic预演播放流畅度 | 长时间线卡顿 | 预加载前后3个Shot+素材缓存+Web Worker |
| Agent API成本 | 单项目可能消耗大量Token | 分级模型策略+Haiku优先+缓存结果 |
| 三段式学习成本 | 用户不知道该在哪个Stage | 自动建议过渡+进度指示+引导教程 |

### 29.2 P1 — 重要风险

| 风险 | 影响 | 缓解 |
|------|------|------|
| 空间语义推断准确度 | 参考图放在错误位置导致误关联 | 明确的关联指示器+手动确认+撤销 |
| Cinema Lab参数→Prompt转换质量 | 参数不能准确反映到AI生成 | 预设模板+人工校验+持续迭代Prompt |
| Suno/Udio API稳定性 | 音乐生成失败 | 重试+多provider备选+本地音效库fallback |
| 剪映Draft格式兼容 | 剪映更新导致格式变化 | 版本检测+格式适配层+降级输出(纯视频+SRT) |
| 资产锁定传播链 | 级联更新导致大量重新生成 | 批量处理+用户确认+增量更新 |
| Color Script↔灯光联动复杂 | 双向绑定状态同步问题 | 单向数据流+明确优先级(用户操作>自动) |

### 29.3 P2 — 一般风险

| 风险 | 影响 | 缓解 |
|------|------|------|
| 知识引擎建议质量 | 建议不相关→用户关闭 | A/B测试+相关性评分+用户反馈学习 |
| Marketplace内容质量 | 低质量模板泛滥 | 审核机制+评分排序+官方推荐 |
| 桌面端Tauri兼容 | Win/Mac差异 | 先Web稳定→桌面适配+Tauri IPC抽象层 |
| i18n翻译质量 | 专业术语翻译不准 | 术语表+专业校对+社区贡献 |
