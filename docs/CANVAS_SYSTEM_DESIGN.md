# 影视创作画布系统设计稿

> 版本：2.0
> 日期：2026-03-06
> 画布定位：`Tapnow/TapFlow` 式可执行无限画布工作台，吸收 `Figma/Miro` 的协作优势
> 关联文档：`docs/TRI_STAGE_CREATIVE_UX.md`、`docs/WORKFLOW_SCHEMA.md`、`docs/CANVAS_API_SPEC.md`、`docs/CANVAS_PHASE1_TASKS.md`

---

## 1. 设计目标

画布系统不是通用白板，而是影视生产工作台。
它要解决的是“如何把故事结构、镜头规划、生成链路、结果评审和回写动作，放在一个清晰、可运行、可协作的空间里”。

因此画布系统必须同时满足：

- 可执行：节点和连线都具有明确契约
- 可观察：每一步执行状态、结果和风险可见
- 可回写：结果可以安全写回项目真相层
- 可协作：支持评论、评审、共享、存在感
- 可控复杂度：默认不把所有高阶能力暴露给所有人

---

## 2. 核心定位

### 2.1 不是自由白板

画布默认不以灵感发散为目标，不鼓励空白起步，不鼓励任意对象摆放，不鼓励无约束自由连线。

### 2.2 是可执行生产画布

画布的核心对象是：

- 工作流模板
- 可运行节点
- 类型化连线
- 结果工件
- 回写动作
- 评论与评审锚点

### 2.3 结果导向优先于参数导向

用户首先应该看到的是“这一步产出了什么、是否可用、能否回写”，而不是一上来面对大量复杂参数。
参数应渐进打开，且由角色、版本和风险级别共同控制。

---

## 3. 设计原则

### 3.1 模板优先

- 默认从模板创建工作流
- 模板应覆盖主流影视生产路径
- 空白画布只对高阶版本开放

### 3.2 结果优先

- 节点卡默认展示结果摘要
- 结果卡可直接触发比较、回写、复用、评论
- 结果应比参数更醒目

### 3.3 局部重跑优先

- 单节点重跑
- 从当前节点继续向后重跑
- 分支候选比较
- 尽量避免整链重跑

### 3.4 回写显式化

- 所有回写都需要明确目标对象
- 所有回写都要能预览差异
- 所有回写都要可撤回或保留版本痕迹

### 3.5 协作增强而非协作主导

- 评论、框选、Presence、评审模式属于增强层
- 不能让协作装饰压过生产对象本身

### 3.6 复杂度按版本渐进释放

- `Normal`：只见结果与推荐动作
- `Canvas`：可见标准模板和有限编辑
- `Hidden`：开放高级分支与策略
- `Ultimate`：开放实验与系统调试

## 4. 信息架构

### 4.1 页面结构

- 顶部：工作流标题、阶段切换、保存状态、运行状态、分享、协作
- 左侧：模板库、节点库、资产树、Scene/Shot 导航
- 中间：无限画布
- 右侧：节点检查器、结果面板、回写面板、历史面板
- 底部：运行控制台、事件流、告警摘要

### 4.2 画布对象层次

1. 工作流层：表示一个完整编排单元
2. 分组层：表示章节、Scene、子任务、评审区域
3. 节点层：表示可执行任务单元
4. 工件层：表示图片、视频、文本、Prompt、报告、时间线片段等结果
5. 协作层：表示评论、Presence、批注、分享权限

### 4.3 默认视觉层级

- 一级强调：节点状态与结果可用性
- 二级强调：推荐路径与当前执行焦点
- 三级强调：协作评论与附加说明
- 最低强调：高级参数与调试信息

---

## 5. 节点体系

### 5.1 节点分组

- `input`：输入节点
- `analysis`：分析节点
- `creative`：创作节点
- `director`：导演节点
- `visual`：视觉节点
- `generation`：生成节点
- `quality`：质检节点
- `output`：输出节点
- `control`：控制节点

### 5.2 Phase 1 核心节点

- 小说输入
- 剧本输入
- Scene 输入
- Beat 提炼
- Scene 拆镜
- 角色锚点
- 场景锚点
- 风格板
- Prompt 组装
- 镜头图生成
- 镜头视频生成
- 字幕草案
- Animatic 输出
- 回写确认

### 5.3 节点卡展示字段

每个节点卡默认展示：

- 名称
- 类型
- 当前状态
- 输入摘要
- 输出摘要
- 最近一次运行时间
- 候选数量
- 推荐下一步动作

高级参数通过右侧检查器展开，不在默认卡面铺开。

---

## 6. 连线与约束

### 6.1 连线规则

- 连线必须通过类型检查
- 不允许形成非法环
- 高风险节点前必须有必要前置输入
- 不允许直接把不兼容工件写入下游节点

### 6.2 默认推荐布局

- 输入在左，输出在右
- 分析与导演节点位于中前段
- 生成与质检节点位于中后段
- 输出与回写位于最右
- Frame 用于标示章节、Scene 范围、评审范围

### 6.3 自由度控制

- `Normal`：无自由连线
- `Canvas`：有限自由连线，仍受模板和类型约束
- `Hidden`：开放高级分支和更多连接方式
- `Ultimate`：开放嵌套与实验性编排

---

## 7. 协作增强层

### 7.1 吸收自 Figma/Miro 的能力

- 多人 Presence
- 评论锚点
- Frame 与区域命名
- 分享模式
- 审批状态标记
- 变更高亮

### 7.2 协作模式

- 编辑模式：可执行、可改动
- 评审模式：只读、可评论、可批注
- 展示模式：用于汇报或对外分享

### 7.3 评论对象

评论可以挂载在：

- 整体工作流
- Frame 区域
- 单节点
- 单个工件结果
- 回写动作

## 8. 版本差异

| 能力 | `Normal` | `Canvas` | `Hidden` | `Ultimate` |
|---|---|---|---|---|
| 模板工作流 | 是 | 是 | 是 | 是 |
| 单节点重跑 | 限量 | 是 | 是 | 是 |
| 自由连线 | 否 | 受限 | 是 | 是 |
| 条件节点 | 否 | 否 | 是 | 是 |
| 节点嵌套 | 否 | 否 | 部分 | 是 |
| 调试视图 | 否 | 否 | 部分 | 是 |
| 实验模式 | 否 | 否 | 否 | 是 |

---

## 9. Phase 1 可开发范围

### 9.1 必做

- 模板驱动工作流创建
- 节点运行状态
- 节点结果卡
- 单节点重跑
- 回写确认
- 评论锚点
- Scene/Shot 导航联动
- 预演台发送入口

### 9.2 暂不做

- 节点嵌套
- 复杂条件分支
- 多级审批流
- 复杂实验流
- 高级数据看板

---

## 10. 最终结论

影视创作画布必须是一个“以结果和执行为中心”的生产系统。
它的正确形态不是自由白板，而是 `Tapnow/TapFlow` 式可执行无限画布；协作能力只作为增强层叠加其上。
