# 多机位全景操作指南

## 概述

多机位全景功能允许在**同一张 360° 全景图**上设置多个虚拟摄影机位。每个机位不仅有独立的视角 (yaw/pitch/fov)，还支持**相机位移**和**投影偏移（移轴）**，从而产生不同的透视畸变效果——类似电影场景中的多机位拍摄。

---

## 核心概念

| 术语 | 说明 |
|------|------|
| **全景图** | 一张 equirectangular 格式的 360° 球面图像 |
| **点位 (ViewPoint)** | 一个虚拟摄影机位，包含 8 个参数 |
| **偏航 (Yaw)** | 水平旋转角度，-180° ~ 180° |
| **俯仰 (Pitch)** | 垂直仰俯角度，-90° ~ 90° |
| **视场角 (FOV)** | 镜头视野宽度，20°(长焦) ~ 100°(广角) |
| **相机位移 (posX/Y/Z)** | 相机在球内的空间偏移，-150 ~ 150。偏离球心后，近处物体变大、远处变小，产生视差效果 |
| **投影偏移 (projOffset)** | 类似移轴镜头 (Tilt-Shift)，改变透视汇聚点。水平/垂直各 -50% ~ 50% |

### 三层控制维度

```
第1层: 视角控制  →  yaw (水平旋转) + pitch (垂直仰俯)   → 相机朝哪看
第2层: 焦距控制  →  fov (视场角)                         → 广角还是长焦
第3层: 机位控制  →  posX/Y/Z (位移) + projOffsetX/Y (移轴) → 在哪拍 + 怎么拍
```

## 操作流程

### 1. 准备全景图

在资产库中上传或生成 equirectangular 格式的 360° 全景图，关联到 Location。

### 2. 打开全景查看器

在画布中点击 **场景背景 (SceneBG)** 节点的卡片。

### 3. 基本导航

- **鼠标拖拽**: 旋转视角 (yaw + pitch)
- **滚轮**: 调整 FOV（缩放效果）

### 4. 进入编辑模式

在 SceneBGNode 卡片上**右键单击**，卡片标题旁出现"编辑模式"标签。再次点击卡片打开全景查看器进入编辑模式。

### 5. 添加机位

**方式一：点击画面空白处**
1. 在编辑模式下，点击全景画面中的任意空白处
2. 弹出命名对话框，输入机位名称（如"大厅正面"）
3. 确认后，以当前完整相机状态（含位移和偏移）创建机位

**方式二：保存当前视角按钮**
1. 先调整好视角、位移、偏移到满意的状态
2. 点击底部 **"+ 保存点位"** 按钮或左侧面板的 **"+ 保存当前视角"**
3. 输入名称确认

### 6. 调整相机位移（关键功能）

添加机位后，在左侧面板中选中活动机位，面板会展开**编辑滑块**：

#### 相机位移 (X / Y / Z)
- **X 滑块**: 左右移动相机（-150 ~ 150）
- **Y 滑块**: 上下移动相机
- **Z 滑块**: 前后移动相机
- 拖动滑块时**实时预览**，画面立即变化
- 效果：偏离球心后，近景和远景的比例关系改变，产生**视差**

#### 投影偏移 / 移轴 (H / V)
- **H 滑块**: 水平投影偏移（-50% ~ 50%）
- **V 滑块**: 垂直投影偏移
- 效果：透视汇聚点偏移，类似**移轴镜头**，可纠正或夸张建筑透视

### 7. 切换机位

- 点击左侧**机位列表**中的名称
- 或点击球面上的**机位标记**（带标签的圆点）
- 相机以 300ms 平滑动画过渡（位置、方向、FOV、偏移全部插值）

### 8. 截图

1. 切换到目标机位
2. 点击底部 **"截图"** 按钮
3. 系统以 1920×1080 分辨率截取当前视角（含位移和移轴效果）
4. 截图自动上传，下游节点收到更新

### 9. 退出

点击 **关闭** 按钮或按 ESC。再次右键 SceneBGNode 可退出编辑模式。

## 节点卡片信息

SceneBGNode 卡片上显示：

- **左上角**: 点位数量（如 "3 点位"）
- **右上角**: 状态指示灯（蓝=运行中，绿=成功，红=错误）
- **底部叠加层**:
  - `偏航°` `俯仰°` `FOV°` — 基础视角信息
  - 点位标签 — 当前活动机位名称
  - **位移** 标签 — 当该机位有相机位移时显示（紫色）
  - **移轴** 标签 — 当该机位有投影偏移时显示（黄色）
  - **已截图** 标签

## 后端 API

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/projects/{pid}/locations/{lid}/viewpoints` | 列出所有机位 |
| POST | `/api/projects/{pid}/locations/{lid}/viewpoints` | 创建机位 |
| PUT | `/api/projects/{pid}/locations/{lid}/viewpoints/{vp_id}` | 更新机位 |
| DELETE | `/api/projects/{pid}/locations/{lid}/viewpoints/{vp_id}` | 删除机位 |

**完整请求体示例 (POST)**:
```json
{
  "label": "大厅正面机位",
  "yaw": 45.0,
  "pitch": -10.0,
  "fov": 60.0,
  "pos_x": 80.0,
  "pos_y": 0.0,
  "pos_z": -50.0,
  "proj_offset_x": 10.0,
  "proj_offset_y": -5.0,
  "is_default": true
}
```

新增字段均有默认值 `0.0`，完全向后兼容。

## 技术要点

- **相机位移原理**: 默认相机在球心 (0,0,0)，位移后偏离球心，球面各方向距离不再相等，自然产生近大远小的视差效果
- **投影偏移原理**: 通过 `camera.setViewOffset()` 偏移投影平面中心，改变透视汇聚方向，等效于移轴镜头
- **截图保真**: 截图时保持当前完整相机状态（位置+方向+FOV+投影偏移），所见即所得
- **平滑过渡**: 机位切换时对 position、target、fov、projOffset 全部进行 300ms ease-in-out 插值动画
- **向后兼容**: 所有新字段 optional，默认 0，老数据无需迁移
