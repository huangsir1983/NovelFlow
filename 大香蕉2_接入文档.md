# 大香蕉2 当前 API 接入文档

更新时间：2026-03-09
适用版本：当前线上正式版
适用路径：`/nanobanana`

## 1. 当前接入方式结论

当前大香蕉2已经不是旧的 Gemini `generateContent` 直连模式。

当前线上真实接入方式是：

- 上游协议：OpenAI Chat Completions 兼容协议
- 请求路径：`POST {BASE_URL}/v1/chat/completions`
- 认证方式：`Authorization: Bearer <API_KEY>`
- 响应方式：`stream: true` 的 SSE 流式返回

也就是说，如果上游要兼容当前网站，必须兼容这一套，而不是旧的：

- `POST /v1beta/models/...:generateContent`

## 2. 后台可配置项

当前后台 `渠道管理（新） -> nanobanana` 可配置：

- `URL`
- `接口密钥`
- `模型前缀`
- `1K / 2K / 4K` 是否启用与积分价格
- 图像比例开关
- 参考图压缩单边最大限制

前端页面不需要改。

只要上游接口兼容，下游这边只需要：

- 替换 `URL`
- 替换 `密钥`
- 按需要改 `模型前缀`

即可直接使用。

## 3. 当前模型映射规则

### 3.1 模型前缀

后台释放出的字段叫：`模型前缀`

当前默认值：

- `gemini-3.1-flash-image`

这个前缀可以替换成别的模型族，例如：

- `gemini-2.5-flash-image`
- `gemini-3.0-pro-image`
- `imagen-4.0-generate-preview`
- `gemini-3.1-flash-image`

### 3.2 图像比例映射

当前只会发送这 5 个比例：

- `16:9 -> landscape`
- `9:16 -> portrait`
- `1:1 -> square`
- `4:3 -> four-three`
- `3:4 -> three-four`

### 3.3 清晰度映射

当前只保留这 3 档：

- `1K -> 无后缀`
- `2K -> -2k`
- `4K -> -4k`

`3K` 当前已经删除，不会再发送。

### 3.4 最终模型名拼接规则

最终传给上游的 `model` 规则是：

`<模型前缀>-<比例后缀><清晰度后缀>`

示例：

- 前缀 `gemini-3.1-flash-image`
- `16:9 + 1K` -> `gemini-3.1-flash-image-landscape`
- `16:9 + 2K` -> `gemini-3.1-flash-image-landscape-2k`
- `9:16 + 4K` -> `gemini-3.1-flash-image-portrait-4k`
- `1:1 + 1K` -> `gemini-3.1-flash-image-square`
- `4:3 + 2K` -> `gemini-3.1-flash-image-four-three-2k`
- `3:4 + 4K` -> `gemini-3.1-flash-image-three-four-4k`

## 4. 当前请求协议

### 4.1 请求地址

```http
POST {BASE_URL}/v1/chat/completions
```

### 4.2 请求头

```http
Authorization: Bearer <API_KEY>
Content-Type: application/json
```

### 4.3 固定参数

当前网站会固定发送：

- `stream: true`

## 5. 文生图请求格式

当用户没有上传参考图时，当前网站发送的是：

```json
{
  "model": "gemini-3.1-flash-image-landscape",
  "messages": [
    {
      "role": "user",
      "content": "一只橘猫坐在窗台上，午后阳光，写实风格"
    }
  ],
  "stream": true
}
```

注意：

- 纯文生图时，`messages[0].content` 是**纯字符串**
- 上游必须兼容这种写法

## 6. 图生图 / 多图参考请求格式

当用户上传参考图时，当前网站发送的是：

- `messages[0].content` 变成数组
- 图片按 `image_url` 传递
- 最后再追加一段 `text`

示例：

```json
{
  "model": "gemini-3.1-flash-image-square-2k",
  "messages": [
    {
      "role": "user",
      "content": [
        {
          "type": "image_url",
          "image_url": {
            "url": "data:image/jpeg;base64,/9j/4AAQSk..."
          }
        },
        {
          "type": "image_url",
          "image_url": {
            "url": "data:image/png;base64,iVBORw0KGgo..."
          }
        },
        {
          "type": "text",
          "text": "保留主体姿态和服装，改成电影感海报风格"
        }
      ]
    }
  ],
  "stream": true
}
```

注意：

- 当前网站最多支持 `7` 张参考图
- 图片 URL 传的是完整：
  - `data:image/...;base64,...`
- 不是文件上传，也不是外部 URL 必传

## 7. 上游返回要求

### 7.1 必须是 SSE 流式返回

响应头至少应类似：

```http
Content-Type: text/event-stream
```

### 7.2 当前下游如何解析流

当前网站会读取每一条 SSE `data:`，并解析：

- `choices[0].delta.content`
- `choices[0].delta.reasoning_content`

最后把整段内容拼起来，再从里面提取图片结果。

### 7.3 当前下游支持的最终结果格式

当前网站能解析以下几种格式，任意一种都可以：

#### 格式 A：Markdown 图片链接

```text
![Generated Image](https://example.com/xxx.png)
```

#### 格式 B：Markdown + data URL

```text
![Generated Image](data:image/png;base64,iVBORw0KGgo...)
```

#### 格式 C：纯直链

```text
https://example.com/xxx.png
```

#### 格式 D：纯 data URL

```text
data:image/png;base64,iVBORw0KGgo...
```

#### 格式 E：JSON 文本里带 url

```json
{"url":"https://example.com/xxx.png"}
```

### 7.4 当前推荐的最稳返回方式

推荐上游最终直接回以下两种之一：

1. `![Generated Image](https://...)`
2. `data:image/png;base64,...`

这样当前网站可以直接解析，不需要改代码。

### 7.5 流结束

当前推荐正常输出：

```text
data: [DONE]
```

## 8. 失败处理要求

如果生成失败，不要只返回空流。

建议明确返回失败文本，例如：

```text
Generation failed: <具体原因>
```

否则下游会判定为：

- 无法解析结果
- 或只返回文本无图片

并按失败处理。

## 9. 兼容要求总结

如果上游要做到“下游只换 URL 和密钥即可直接使用”，必须满足以下条件：

1. 支持：
   - `POST /v1/chat/completions`
2. 支持：
   - `Authorization: Bearer <API_KEY>`
3. 支持：
   - `stream: true`
4. 支持纯文生图：
   - `messages[0].content = "字符串"`
5. 支持图生图 / 多图：
   - `messages[0].content = [{type:"image_url"}, {type:"text"}]`
6. 支持模型名按当前规则直接传入：
   - `<prefix>-<ratioSuffix><sizeSuffix>`
7. SSE 最终必须真正返回图片结果
8. 返回结果需兼容以下之一：
   - Markdown 图片链接
   - 纯 URL
   - data URL
   - JSON url
9. 最终建议输出 `[DONE]`

## 11. 一句话结论

当前大香蕉2这条线的真实接法是：

- 下游固定走 `POST /v1/chat/completions`
- 后台通过 `模型前缀 + 比例 + 清晰度` 拼出完整模型名
- 文生图传纯字符串 `content`
- 图生图传 `image_url + text`
- 上游只要兼容这套协议，下游就只需要替换 `URL` 和 `密钥`，不需要再改代码。
